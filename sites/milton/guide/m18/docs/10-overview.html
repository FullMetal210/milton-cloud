Milton-core-API

Following diagram shows the main components of Milton API.

HTTPManager: this class is core of Milton API. It handles the initialization of Resource factory, Resource Handler and Authentication Handler during the startup. It initializes the default resource handler if none was specified. Upon receiving the request, it invokes the processing method of the Resource handler. It is the Engine of the Milton API.

Security, Filter: yet to be completed

Request & Repsonse : represents the WebDAV Request object and Response objects and defines the abstract methods.

Resource Factory: It identifies an instance of a resource at the given url and on the given host

Resources: This is an interface and notable sub-classes are Getable, Postable, putable etc. These Resource Classes serve the corresponding requests. For Example, Getable Resource serves the GET requests. Explanation and usage is covered in details below.

Handlers: it takes the request, response and resource as an input and process the resources appropriately. For example, PutHandler creates or overwrites the resource. GetHandler returns the content by invoking HTTPResponse Handler.

Milton-Servlet-API

MiltonServlet: Main Servlet and act as a Gateway to the Milton API. Once Milton Servlet receives the request, it hands it over to the ServletHTTPManager.

ServletHTTPManager: Just extends the normal HttpManager to support initializing filters from servlet init params.

SpringAwareMiltonServlet: This spring aware servlet will load the spring context from a classpath resource named /applicationContext.xml. Please refer the Javadoc for more details about the implementation.

ResourceFactoryFactory: This can be used to configure the Resource Factory. Please refer the Javadoc for more details about the implementation.
