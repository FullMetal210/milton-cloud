Scenario:

we will consider the following real time scenario. There is a company ABC and it has departments like HR, Finance, IT, Sales. Company needs a effective document management system i.e. Kind of central repository for documents. Company is now hiring for various departments. So we will build a system where users can upload, download and manage the resumes among the various departments. We use database for storing the objects. This example runs on tomcat and uses Hibernate for persistence layer. You can find the link to download the entire project in the Reference section.

 

Hence our application needs to provide the ability to

  • Create a new department
  • Remove the existing department
  • Rename the department
  • Upload the document
  • Download the document
  • Lock / unlock the document for overwrite prevention
  • Check who updated the document recently and when
  • Rename the document
  • Copy the document from one department to another

 

As you aware WebDAV provides ability to map the drive or mount the drive so that user can work as if it were on the local system. They can use the native clients like windows explorer. We will use the existing database to store the details like departments and documents.

We will build the application step by step. We will to use “Hibernate” for persistence.
