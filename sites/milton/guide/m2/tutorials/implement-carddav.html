To implement CardDav support in your application you must:

  1. Implement AddressResource to represent a contact. Example .
  2. Implement AddressBookResource and ReportableResource on your contacts collection resources. Example

Note that the AddressResource interface requires you to return contact information as VCard formatted text. Caldav contacts will be created when clients do a PUT with information formatted as VCard text. An application may choose to simply persist the text and return it as required, or you may choose to parse it and store it in structured form, reconstituting the VCard on demand, or you might store the text and parsed data.

The Cardme library is perfect for parsing and formatting VCard text, although milton has no dependency on it. Milton does provide a forked copy of Cardme because at the time of writing there was otherwise no pre-built stable latest version. This is available in the milton maven repository with the following vector:

        <dependency>
            <groupId>io.milton</groupId>
            <artifactId>cardme</artifactId>
            <version>1.0.0</version>
        </dependency>

However, the official Cardme distribution should be preferred long term. Implementors are encouraged to start with the official cardme distribution and use the milton one if problems are encountered.
